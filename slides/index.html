Let's begin
Electricity = magic
Basic building block
- Basic building blocks of any circuit are components, you've heard of resistors, capcitors and transistors - Also will be building using Integrated Circuits (ICs) - - In software you *could* write all the functionality from scratch, but more complex and more likely to get something wrong - Same thing in electronics. Many Integrated Circuits (ICs) exist that perform complex or common tasks in a neat package. - charging a battery cell, converting from one protocol to another or complex sensors. - Microcontroller = IC = mini computer
What we'll need
- Making a LED matrix means we need LEDs, a LED driver and a microcontroller - Power input and probably some passive components - Resistors can reduce the voltage to a component or provide a connection to power or ground resulting in a consistent input to a component. - Capacitors absorb spikes in voltage and provide a easy supply of current when needed by a component
Shopping for components
- So many suppliers and specifications and parameters - Figure out your main requirements, then sort by cost/availability - It’s actually quite fun once you get into it
- 3 main parts, USB-C connector, LED driver and microcontroller (and LEDs) - If you're into AWS you might find this one easier - Needed to get familar with KiCad through youtube videos. Helped with schematic standards
How to connect
PCBs are the way
- Components are soldered to pads on the PCB and traces connect between pads
- PCBs can have multiple layers allowing you to have double sided boards or boards with internal layers for more routing, or power and ground connection - To connect different layers, holes are drilled through the board and filled with copper.
- Choose some board shape/dimensions
- Each component on the schematic will have a “footprint” which defines the copper pad markings it needs to be placed on the board. - Place your components but they may need to be rotated or moved depending on how you want to connect them
- Can place components on either side of the board
- Lots of tools to help you position things
- To connect components on different layers use vias
- To provide components with ground and power you can tap into the relevant plane with a via
- Final board design - Run automated checks, not comprehensive but pretty good for basic mistakes. Definitely not as good as type checking and unit tests
- V1 went for symmetry, didn't play nicely with the USBC components - Missing capacitor for the LED driver - Doesn't utilise most space - Could have made it work
- V2 went for vibes, still coulnd't fit the USBC resistors on the same side
- V3 made mistakes with LED footprint and connections to ground instead of power - V4 worked perfectly
